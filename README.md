# 🐾 CatChat

یک چت‌باکس ساده، آنلاین و چند کاربره؛ با اکانت‌سازی، ریپلای، حذف پیام، ضدِ اسپم و سانسور کلمات.

این فایل، راهنمای کامل قدم‌به‌قدم پروژه‌ست: از فهمیدن کدها تا بالا آوردنش روی اینترنت.

---

## ۱) این سایت دقیقاً چیکار می‌کنه؟

- هر کاربر با یک **نام کاربری منحصربه‌فرد + رمز عبور** اکانت می‌سازه و وارد میشه.
- پیام‌ها برای همه‌ی کاربرای واردشده نمایش داده میشه و هر ۳ ثانیه لیست پیام‌ها خودکار تازه میشه.
- میشه روی هر پیام «پاسخ» زد؛ پیام جدید یک قسمت کوچیک از پیام اصلی رو بالای خودش نشون میده.
- هر کاربر فقط می‌تونه **پیام‌های خودش** رو حذف کنه.
- یک نام کاربری که تو از قبل مشخص می‌کنی، **ادمین** حساب میشه و می‌تونه هر پیامی از هر کسی رو حذف کنه.
- بین دو پیام متوالی از یک کاربر، حداقل ۳ ثانیه فاصله لازمه (ضدِ اسپم).
- حتی بعد از اون ۳ ثانیه، اگه پیام جدید *دقیقاً* مثل آخرین پیام خودِ همون کاربر باشه، رد میشه.
- کلماتی که خودت مشخص می‌کنی، خودکار با `***` جایگزین میشن.
- دیتابیس روی **Supabase** (رایگان) نگه‌داری میشه؛ یعنی اکانت‌ها و پیام‌ها همیشه می‌مونن، حتی وقتی سایت روی هاست رایگان می‌خوابه یا دوباره دیپلوی میشه.

## ۲) ساختار فایل‌ها

```
catchat/
├── server.js          ← بک‌اند: کل منطق سایت و API اینجاست
├── censoredWords.js    ← لیست کلمات سانسوری (خودت پرش می‌کنی)
├── package.json        ← لیست کتابخونه‌های موردنیاز پروژه
├── .env.example         ← نمونه‌ی فایل تنظیمات محرمانه
├── .gitignore
└── public/               ← فرانت‌اند (چیزی که توی مرورگر کاربر باز میشه)
    ├── index.html         ← ساختار صفحه
    ├── style.css           ← ظاهر و رنگ‌بندی
    └── script.js            ← رفتار صفحه (فرستادن/گرفتن پیام و ...)
```

## ۳) کدها چطور کار می‌کنن (توضیح فایل به فایل)

### server.js
اینجا با **Express** یک سرور ساخته شده که:
- فایل‌های داخل `public/` رو مستقیم برای مرورگر کاربر می‌فرسته.
- با یک پروژه‌ی **Postgres روی Supabase** صحبت می‌کنه (از طریق کتابخونه‌ی `pg`) و در صورت نیاز، دو جدول می‌سازه: `users` (کاربرا) و `messages` (پیام‌ها). هر پیام یک `reply_to_id` داره که اگه پاسخ به پیام دیگه‌ای باشه، شماره‌ی همون پیام رو نگه می‌داره.
- رمزهای عبور هیچ‌وقت مستقیم ذخیره نمیشن؛ با کتابخونه‌ی `bcryptjs` هش (رمزنگاری یک‌طرفه) میشن.
- بعد از ورود موفق، یک **JWT token** (یک رشته‌ی امضاشده و امن) به مرورگر کاربر داده میشه. مرورگر این توکن رو توی حافظه‌ی خودش (`localStorage`) نگه می‌داره و برای هر درخواست بعدی (فرستادن پیام، حذف و ...) به سرور نشونش می‌ده تا سرور بفهمه این درخواست از طرف کدوم کاربره.
- تشخیص ادمین بودن، **ذخیره نمیشه**؛ هر بار زنده بر اساس مقایسه‌ی نام‌کاربری با متغیر `ADMIN_USERNAME` محاسبه میشه. یعنی هر وقت اون متغیر رو عوض کنی، همون لحظه اکانت جدید ادمین میشه، بدون نیاز به دستکاری دیتابیس.
- روت `POST /api/messages` سه تا بررسی مهم داره، دقیقاً به ترتیبی که خواسته بودی:
  1. آیا کمتر از ۳ ثانیه از آخرین پیام همین کاربر گذشته؟ → رد میشه.
  2. آیا متن پیام جدید *عیناً* با آخرین پیام همین کاربر یکیه؟ → رد میشه.
  3. سانسور کلمات بد روی متن اجرا میشه و بعد پیام ذخیره میشه.
- روت `DELETE /api/messages/:id` چک می‌کنه که یا صاحب پیام خودِ کاربره، یا کاربر ادمینه؛ در غیر این صورت خطای «اجازه نداری» برمی‌گردونه.

### censoredWords.js
یک آرایه‌ی ساده از کلمات. تابع `censorText` توی `server.js` هر کدوم از این کلمات رو هر جای پیام پیدا کنه، با تعداد برابر ستاره (`*`) عوض می‌کنه. **عمداً این لیست رو خالی گذاشتم** چون انتخاب این‌که چه کلماتی برای جامعه‌ی کاربرای خودت مناسبه، به عهده‌ی خودته؛ فقط کافیه کلمه‌های موردنظرت رو داخل این فایل، هر کدوم بین کوتیشن `' '` و با کاما جدا از هم، اضافه کنی (نمونه‌ش داخل خود فایل نوشته شده).

### public/index.html + style.css + script.js
یک صفحه‌ی تک‌صفحه‌ای (بدون رفرش شدن) هست: اگه لاگین نکرده باشی فرم ورود/ثبت‌نام رو می‌بینی، وگرنه مستقیم چت رو. `script.js` هر ۳ ثانیه از سرور می‌پرسه «پیام جدید هست؟» (به این روش میگن **polling**) و لیست رو به‌روز می‌کنه. متن پیام‌ها همیشه با `textContent` (نه `innerHTML`) نمایش داده میشه تا اگه کسی توی پیامش کد HTML/جاوااسکریپت بنویسه، اجرا نشه و امنیت حفظ بشه.

## ۴) ساخت پروژه‌ی دیتابیس روی Supabase

قبل از اجرای سایت (حتی به‌صورت محلی)، به یه پروژه‌ی Supabase نیاز داری. این کار فقط یک بار لازمه:

1. برو به https://supabase.com و با ایمیل یا اکانت گیت‌هابت ثبت‌نام کن (رایگانه، نیازی به کارت بانکی نیست).
2. از داشبورد، **New Project** رو بزن.
3. یک اسم دلخواه بذار (مثلاً `catchat`)، یک **Database Password** قوی انتخاب کن و **آن رو جایی یادداشت کن** (بعداً لازمش داری)، نزدیک‌ترین Region رو انتخاب کن، و **Create new project** رو بزن.
4. چند دقیقه صبر کن تا پروژه ساخته و آماده بشه (یک نشونه‌ی لودینگ می‌بینی).
5. وقتی آماده شد، بالای صفحه دنبال دکمه‌ی سبزرنگ **Connect** بگرد و بزنش.
6. توی پنجره‌ای که باز میشه، از قسمت **Connection String**، تب **Session pooler** رو انتخاب کن (نه Direct connection و نه Transaction pooler) و لینکی که شبیه این هست رو کپی کن:
   ```
   postgresql://postgres.xxxxxxxx:[YOUR-PASSWORD]@aws-0-xxxxx.pooler.supabase.com:5432/postgres
   ```
7. توی همین لینک، به‌جای `[YOUR-PASSWORD]` همون رمزی که قدم ۳ ساختی رو جایگزین کن. این همون مقداریه که چند خط پایین‌تر، توی `DATABASE_URL` قرارش می‌دیم.

همین! دیگه لازم نیست هیچ جدولی رو خودت دستی بسازی؛ اولین باری که `server.js` رو اجرا کنی، خودش جدول‌های `users` و `messages` رو می‌سازه.

## ۵) اجرای سایت روی سیستم خودت (تست محلی)

### قدم اول: نصب Node.js
اگه از قبل نصب نداری، از سایت رسمی نسخه‌ی **LTS** رو دانلود و نصب کن:
👉 https://nodejs.org

برای اطمینان از نصب، توی همون پنجره‌ی سیاه (ترمینال) بزن:
```
node -v
```
اگه یک شماره نسخه نشون داد، یعنی آماده‌ای.

### قدم دوم: فایل‌های پروژه رو باز کن
فایل زیپی که براَت فرستادم رو توی یک پوشه (مثلاً روی دسکتاپ) اکسترکت/باز کن. بعد توی همون پنجره‌ی سیاه به همون پوشه برو:
```
cd path/to/catchat
```

### قدم سوم: فایل .env رو بساز
از Notepad یک فایل جدید بساز، این چهار خط رو توش بنویس (مقدارهای واقعی خودتو جایگزین کن)، و با اسم دقیق `.env` (با گزینه‌ی «Save as type: All Files») توی پوشه‌ی `catchat` ذخیره‌ش کن:
```
DATABASE_URL=postgresql://postgres.xxxxxxxx:رمز-واقعی-خودت@aws-0-xxxxx.pooler.supabase.com:5432/postgres
JWT_SECRET=یک-رشته-تصادفی-و-طولانی
ADMIN_USERNAME=نام-کاربری-ادمین
PORT=3000
```
برای ساختن یه `JWT_SECRET` تصادفی، توی همون پنجره‌ی سیاه بزن:
```
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

### قدم چهارم: نصب کتابخونه‌ها
```
npm install
```
هیچ‌کدوم از کتابخونه‌های این پروژه نیاز به کامپایل یا ابزار اضافه (مثل Visual Studio) ندارن، پس این قدم باید سریع و بدون خطا تموم بشه.

> اگه با ارور «npm is not recognized» مواجه شدی، یعنی Node.js نصب نیست یا پنجره‌ی ترمینال از قبل باز بوده؛ برگرد به قدم اول و بعد از نصب، پنجره رو کامل ببند و دوباره باز کن.

### قدم پنجم: اجرا
```
npm start
```
اگه توی ترمینال پیام «CatChat روی http://localhost:3000 در حال اجراست» رو دیدی، مرورگرت رو باز کن و برو به:
👉 http://localhost:3000

اگه به‌جاش خطایی درباره‌ی اتصال به دیتابیس دیدی، مطمئن شو `DATABASE_URL` توی `.env` رو درست (با رمز واقعی، نه `[YOUR-PASSWORD]`) کپی کردی.

حالا می‌تونی ثبت‌نام کنی و پیام بفرستی. برای تست چندکاربره، همین آدرس رو توی یک تب دیگه یا یک مرورگر دیگه (یا حالت Incognito) باز کن و با اکانت دوم وارد شو.

## ۶) چطور ادمین بشی

فقط کافیه با همون نام کاربری‌ای که توی `ADMIN_USERNAME` (داخل `.env`) نوشتی، **ثبت‌نام** کنی. همون لحظه، کنار اسمت توی بالای صفحه یک 👑 می‌بینی و دکمه‌ی «حذف» روی همه‌ی پیام‌های همه‌ی کاربرا برات ظاهر میشه.

⚠️ نکته: هر بار فایل `.env` رو عوض می‌کنی، باید سرور رو ری‌استارت کنی (`Ctrl+C` بعد دوباره `npm start`) تا تغییر رو ببینه. اگه بعد از ثبت‌نام مقدار `ADMIN_USERNAME` رو عوض کردی، اکانت‌های قبلی به‌طور خودکار عوض نمیشن؛ باید با نام‌کاربری جدید دوباره وارد بشی (خروج و ورود دوباره، تا توکنت تازه بشه).

بعد از هر تغییر توی این فایل، اگه سرور محلی روشنه باید یک بار متوقفش کنی (Ctrl+C) و دوباره `npm start` بزنی تا تغییرات اعمال بشه. روی هاست آنلاین هم کافیه فایل رو توی گیت‌هاب آپدیت کنی؛ خودش دوباره دیپلوی میشه.

## ۸) پابلیش کردن سایت (رایگان) روی اینترنت

برای این کار از **GitHub** (برای نگهداری کدها) و **Render** (برای اجرای آنلاین سایت) استفاده می‌کنیم؛ هر دو رایگانن و نیازی به کارت اعتباری ندارن. دیتابیس هم که از قبل روی Supabase آماده‌ست، همینه که هست -- سایتِ آنلاین هم به همون پروژه‌ی Supabase وصل میشه.

### قدم اول: ساخت ریپازیتوری روی گیت‌هاب
1. اگه اکانت گیت‌هاب نداری، توی https://github.com یکی بساز.
2. گوشه‌ی بالا سمت راست، روی علامت `+` بزن و **New repository** رو انتخاب کن.
3. یک اسم بذار (مثلاً `catchat`)، حالتش رو **Public** بذار، و **Create repository** رو بزن.
4. توی صفحه‌ای که باز میشه دنبال لینک **uploading an existing file** بگرد (یا از منوی «Add file» گزینه‌ی **Upload files** رو بزن).
5. تمام فایل‌ها و پوشه‌ی `public` رو از پوشه‌ی پروژه‌ت بکش و داخل اون صفحه رها کن (Drag & Drop). **فایل `.env` رو آپلود نکن** (محتویاتش محرمانه‌ست؛ فقط `.env.example` باید آپلود بشه).
6. پایین صفحه دکمه‌ی **Commit changes** رو بزن.

*(اگه با Git از قبل آشنایی داری، می‌تونی به‌جای این روش، از دستورات `git init`, `git add .`, `git commit`, `git push` هم استفاده کنی؛ نتیجه یکیه.)*

### قدم دوم: ساخت سرویس روی Render
1. برو به https://render.com و با اکانت گیت‌هابت ثبت‌نام کن.
2. از داشبورد، **New +** → **Web Service** رو بزن.
3. ریپازیتوری `catchat` که ساختی رو انتخاب و وصل کن.
4. توی فرم تنظیمات:
   - **Build Command:** `npm install`
   - **Start Command:** `npm start`
   - **Instance Type:** گزینه‌ی **Free** رو انتخاب کن.
5. بخش **Environment Variables** رو باز کن و همون سه مقداری که توی `.env` گذاشته بودی رو اینجا هم اضافه کن:
   - `DATABASE_URL` = همون لینک اتصال Supabase (با رمز واقعی)
   - `JWT_SECRET` = همون رشته‌ی تصادفی
   - `ADMIN_USERNAME` = همون نام کاربری ادمین
6. **Create Web Service** رو بزن. چند دقیقه طول می‌کشه تا بیلد و دیپلوی بشه؛ در پایان یک آدرس شبیه `https://catchat-xxxx.onrender.com` بهت میده. همین لینک، آدرس زنده‌ی سایتته که می‌تونی برای دوستات بفرستی 🎉

### نکته‌ی مهم درباره‌ی نسخه‌ی رایگان Render
اگه سایت ۱۵ دقیقه هیچ بازدیدکننده‌ای نداشته باشه، خودکار «می‌خوابه»؛ اولین نفری که بعدش وارد بشه باید حدود یک دقیقه صبر کنه تا دوباره بیدار بشه. این فقط یعنی کندی توی اولین بار -- برخلاف قبل، **دیگه هیچ دیتایی از بین نمیره**، چون دیتابیس روی Supabase جدا از این سرویسه و خودش زنده می‌مونه.

## ۹) نکات امنیتی و محدودیت‌های فعلی (خوبه بدونی)

- رمزهای عبور هش‌شده ذخیره میشن، و گواهی SSL/HTTPS رو خودِ Render به‌صورت خودکار و رایگان برات فعال می‌کنه.
- نسخه‌ی رایگان Supabase: ۵۰۰ مگابایت فضای دیتابیس (برای یه چت متنی، حجم خیلی زیادیه) و اگه **هفت روز کامل** هیچ درخواستی به دیتابیس نره، پروژه موقتاً مکث می‌کنه. هیچ دیتایی پاک نمیشه؛ فقط اولین درخواست بعد از اون مکث، حدود ۳۰ ثانیه طول می‌کشه تا دوباره بیدار بشه. برای یه سایت که هر چند روز یک‌بار حداقل یه بازدید داره (که یعنی تقریباً همیشه)، این محدودیت اصلاً به چشم نمیاد.
- فیلتر کلمات، ساده و بر پایه‌ی «پیدا کردن متن دقیق» هست؛ کسی که خیلی مصمم باشه می‌تونه با فاصله انداختن بین حروف یا اموجی از دستش در بره. برای یک چت‌باکس دوستانه‌ی ساده کفایت می‌کنه، ولی یک فیلتر ضدگلوله نیست.
- گرفتن لیست پیام‌ها هر ۳ ثانیه (polling) برای یک گروه کوچیک کاملاً کافیه؛ اگه یک روز جمعیت سایتت خیلی زیاد شد، می‌شه بعداً به WebSocket (پیام‌رسانی آنی واقعی) ارتقاش داد.
- در حال حاضر امکان تغییر رمز عبور یا حذف کامل اکانت وجود نداره؛ اینا رو می‌شه بعداً به‌عنوان قابلیت جدید اضافه کرد.
